🔐
What is NEVER stored as plaintext
All sensitive values are hashed or derived before DB insertion
❌ Never: User Password
Identity hashes with PBKDF2 automatically. Column PasswordHash stores only the derived hash.
❌ Never: OTP Code
OTP is sent in Gmail only. Column OTPHash = SHA-256(otp). DB breach can't replay OTPs.
❌ Never: Refresh Token (plain in DB)
Plain Guid generated → sent to client via HttpOnly Secure Cookie. SHA-256(guid) hash stored in DB. On validate: server hashes incoming plain token → compares with stored hash.
❌ Never: JWT Secret Key
Not in DB. Read at runtime from JWT_SECRET_KEY env var. Dev fallback: appsettings only with ⚠ warning.
